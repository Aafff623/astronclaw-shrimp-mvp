## 2. 项目定位

> _一句话_：**学术周报自动化 Agent** —— 每周五自动抓取 arXiv + GitHub Trending，AI 排序生成中文周报，并推送到飞书 / 企业微信 / Notion，全程无人值守。

### 2.1 解决的痛点
- **痛点 1**：AI × Web3 方向资讯源分散（arXiv / GitHub / Twitter / 顶会），每周手动追踪耗时 2-3 小时
- **痛点 2**：单纯的"AI 问答"无法持续沉淀，需要**真正的 Agent 工作流**：调度 → 抓取 → 决策 → 执行 → 归档
- **痛点 3**：现有工具要么只解决"抓取"，要么只解决"摘要"，缺少**多渠道分发 + Notion 知识库写入**的闭环

### 2.2 目标用户
- AI × Web3 方向的研究者、博士生、独立开发者
- 需要每周向团队同步行业动态的 Tech Lead
- 个人知识库（Notion）使用者

## 4. 实现方案

### 4.1 核心流程
1. **定时触发** → cron `0 9 * * 5`（每周五 09:00 北京时间）
2. **并行抓取** → arXiv (cs.AI / cs.CL) + GitHub Trending (Python / Solidity)，单 Skill 内并发
3. **AI 排序与摘要** → 讯飞星火按主题相关度打分，取 top 10，输出 JSON Schema
4. **多模态封面** → AstronClaw image_generate 生成本期周报封面
5. **模板渲染** → Handlebars 引擎组装 Markdown 周报
6. **三渠道并行推送** → 飞书交互卡片 + 企业微信 markdown + Notion 数据库写入
7. **失败重试** → 指数退避 3 次，最后失败时告警推送

### 4.2 Skill 设计
唯一 Skill：[`weekly-digest.yaml`](../skills/weekly-digest.yaml)，约 200 行，覆盖：
- 5 步串行流水线（fetch → rank → cover → render → push）
- 内部 step 2 + step 5 用 `type: parallel` 提升执行效率
- 沙箱白名单显式声明 5 个外部域名
- 失败重试 + 异常告警内建

### 4.3 关键代码片段（节选）
```yaml
trigger:
  type: schedule
  cron: "0 9 * * 5"
  timezone: Asia/Shanghai

steps:
  - id: rank
    type: llm
    model: ${env.ASTRONCLAW_DEFAULT_LLM}
    prompt: |
      你是 AI × Web3 周报编辑，对原始条目按相关度排序并写中文摘要...
    output_format: json_schema
    schema: { ... }
```
完整版见 [`skills/weekly-digest.yaml`](../skills/weekly-digest.yaml)。

## 5. 难点与解决

| # | 难点 | 解决方案 |
|---|------|----------|
| 1 | arXiv 返回 Atom XML，需要解析后才能给 LLM | Skill 内 `parse: atom` 字段声明解析器，平台自动转 JSON |
| 2 | LLM 排序输出格式不稳定（自由文本时） | 强制 `output_format: json_schema`，平台保证返回 schema 合规 |
| 3 | 三渠道推送失败概率不同（Notion 偶发 502） | `retry` 配置指数退避 3 次 + `on_failure` 兜底告警 |
| 4 | 沙箱内不允许任意 HTTP 出站 | 在 Skill `security.network_allowlist` 显式声明 5 个域名 |
| 5 | 多模态封面图风格易飘 | 固定 prompt 模板 + `aspect_ratio: 16:9` + 色板限定 |
