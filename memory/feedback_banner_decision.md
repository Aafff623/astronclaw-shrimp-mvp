参赛 MVP 的 Banner 走**方向 B**（科技养虾感）而不是方向 A（极简抽象）。

## 决策

| 方向 | 描述 | 优势 | 风险 |
|------|------|------|------|
| A 极简抽象 | 紫蓝渐变 + 抽象几何 + claw 隐含轮廓 | 保守安全 | 评审记不住，与其他项目长得像 |
| **B 科技养虾感** ✅ | 半抽象机械虾轮廓 + 神经网络线 + amber 高光 | **有记忆点**，IP 强 | 风险大，做不好像小学生作业 |

**Why**：评审一天看 50 个项目，能记住的是「那个有发光机械虾的项目」，不是「那个紫蓝渐变抽象图」。
赛事 IP 摆在那（"养虾"），完全避开是浪费，但直接画卡通虾会掉档次 —— 半抽象机械虾是中间地带。

## 生成路径

- **首选**：GPT-Image-2（用户自己生成，质量稳定可控）
- **备选**：Replicate Flux Schnell（本次因余额不足未使用）
- **占位**：手工 SVG（紧急情况下保仓库不破图）

## 实际效果（用户生成版）

- 手写体 "AstronClaw" 主标 + 副标题 `Agentic Systems · Tool Use · Memory · Deployment · Observability`
- 副标题正好是 Agent 设计圈五要素 → 与 weekly-digest Skill 五要素映射完美对齐
- 深空背景 + 粒子龙虾 + 紫蓝色板 + amber 局部高光

## How to apply

视觉资料决策时：

1. **记忆点优先于安全感**，但不能突破档次
2. **赛事 IP 是资产**，完全避开是浪费
3. **副标题是隐藏的章节标题**，banner 文字与 README 结构应形成闭环
4. **手工 SVG 是最后兜底**，不是首选 —— 真品图永远比矢量占位强
