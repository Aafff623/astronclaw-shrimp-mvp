2026-05-22 确定 MVP 方向：**weekly-digest（学术周报自动化 Agent）**。

## 核心判断

AstronClaw 平台的卖点是「**端云一体 + RPA 执行 + 沙箱安全 + 多模态**」。
一个不调用 RPA、不写第三方系统、不做调度的 Demo，等于给评审递了把「这平台没必要」的刀。

5 个通用候选的问题：

| 候选 | 致命问题 |
|------|----------|
| 论文助手 | 烂大街，AI Demo 默认选项，评审审美疲劳 |
| 待办 Agent | 不需要 AstronClaw 也能做，反证平台没必要 |
| 简历优化 | 纯对话，零执行能力，浪费 RPA / 沙箱卖点 |
| 微信自动回复 | 场景模糊 + 易触合规红线（骚扰举报） |
| 解题虾 | 多模态加分但仍是问答，不是 Agent |

**Why**：评审看的不是"AI 能聊天"，是"AI 真的能干活"。

## weekly-digest 为什么能打

5 步流水线 `fetch → rank → cover → render → push` 完整闭环：

1. **闭环完整**：抓取 → 决策 → 多渠道执行 → 写入第三方系统 —— 把 AstronClaw 全部卖点用上了
2. **真实场景**：每个研究者 / 开发者都有这个痛点，评审自己可能也想用
3. **可复现演示**：cron 触发后 60 秒内有真实推送到飞书，比对话框炫多了
4. **延伸性强**：周报模板能换（行业资讯、招聘、价格监控），一看就懂"架构能复用"

## How to apply

下次设计 Agent Skill 时，用以下问题筛选方向：

1. 是否**必须用**到平台核心能力？（不能用任意 LLM 替代）
2. 是否有**真实执行**动作？（不只是问答）
3. 是否能**演示闭环**？（60 秒能让评审看到完整效果）
4. 是否**避开合规红线**？（推送/抓取/写入都在白名单内）
5. 是否**架构可复用**？（同一套 Skill 换数据源就能换场景）

四个 yes 以上才值得做 MVP。
